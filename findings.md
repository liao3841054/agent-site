# agent-site vs sanwan.ai 对标分析报告

> 分析时间：2026-03-08
> 分析人：engineer（技术专家）
> 目标：找出 agent.nanopromptlab.com 与 sanwan.ai 的功能与技术差距，输出可执行开发待办清单。

---

## 一、sanwan.ai 功能观察

### 1.1 导航结构

sanwan.ai 共有 6 个页面：
- **首页**（index.html）— 核心品牌展示
- **日记**（diary.html）— 养成日记列表（Day 1~25+）
- **文章**（articles.html）— 额外的深度文章页（agent-site 没有！）
- **科普**（science.html）— 原理科普
- **技能**（skills.html）— 技能商店（40+个独立技能页）
- **EasyClaw**（easyclaw.html）— 产品领养/下载引导页（agent-site 没有！）
- **多语言支持**：导航栏有 EN / JA / DE 三个语言切换按钮（agent-site 没有！）

### 1.2 首页功能模块

#### Hero 区
- 大标题 + 副标题（故事驱动：骨折 CEO 养 AI 龙虾）
- **带封面图轮播**：每条日记都有 AI 生成插画，首页上方轮播展示（大图！每张图都是独立插画）
- 左右翻页按钮（`‹` `›`）+ 底部分页点（支持 20+ 条日记的分页导航）
- 轮播每项：**图片 + Day 编号 + 日记标题**（视觉冲击力强）

#### 故事梗概数字卡片
- 两栏布局：左"📖 故事梗概"、右"🔥 媒体效应"
- 每条带 emoji 图标（非纯数字，有更强叙事感）
- 数据包括：条消息、字对话、Agent 数、技能数、直播观看、文章数等

#### CTA 区（首页中段）
- 双按钮：「📖 从 Day 1 看起」+「🦞 领养同款三万」
- **领养按钮直接导向 EasyClaw 产品页**（商业化！）

#### "龙虾是什么"科普模块
- 4 大能力：超级全能、永不休息、自我进化、团队裂变
- **每个能力有锚点链接 `#day6`** 直指具体日记（深度链接）

#### 技能商店预览
- 6 个技能包卡片
- **底部有「5000+ 社区技能 (ClawHub)」外部链接**（生态！）

#### 留言板
- **GitHub Discussions + giscus 嵌入**（社区互动，agent-site 完全没有）

#### 底部
- 页脚带社媒链接：EasyClaw / GitHub / X（Twitter）

### 1.3 日记页（diary.html）

- 按最新日期排序的日记列表
- **每条日记有封面图**（AI 生成插画，16:9）
- 日记标题具有叙事感（如"老板过生日，AI独自值守"）
- 持续更新（Day 25+ 已有）

### 1.4 科普页（science.html）

- 5 章结构，层层递进
- 第三章有**动态数字动画**（CountUp，数字从 0 到目标值滚动）
- **比 agent-site 更有吸引力的公式/对比展示**（AI 用具体对比图解释概念）
- 末尾有留言板

### 1.5 技能商店（skills.html）

- **40+ 个独立技能页**（每个技能有自己的 HTML 页面，如 `/skills/web-search.html`）
- 技能带标签：`官方⭐`、`必装🔥`、`热门`、`社区`
- 包含来自 ClawHub 社区的技能
- 技能卡片点击进入详情页
- 有技能分类/标签体系

### 1.6 EasyClaw 页（easyclaw.html）

- 产品下载/领养引导：本地版 + 云端版
- 常见问题（FAQ）模块
- 对比 ChatGPT 的差异说明
- 强调"完全免费"、"开源透明"

### 1.7 技术特征

| 维度 | sanwan.ai |
|------|-----------|
| 技术栈 | 纯 HTML/CSS/JS，无框架 |
| 动效 | 轮播翻页动画、CountUp 数字、hover 效果 |
| 响应式 | 是（移动端适配） |
| 多语言 | 是（EN/JA/DE 按钮切换） |
| SEO | 每页独立 title + description |
| 社区互动 | giscus 留言板（GitHub Discussions） |
| 内容更新 | 持续，Day 25+ 有新内容 |
| 插画 | 每条日记 AI 生成封面图（大图轮播） |
| 商业化 | EasyClaw 产品页 + 领养 CTA |
| 外部生态 | ClawHub（5000+ 社区技能），easyclaw.com，easyclaw.work |
| 深度链接 | `diary.html#day6` 等锚点 |

## 二、agent-site 现状审计

### 2.1 页面概况

| 页面 | 状态 | 说明 |
|------|------|------|
| index.html | ✅ 完整 | Hero + 统计卡片 + 4能力Tab + 技能预览 + CTA |
| diary.html | ⚠️ 基础可用 | 有筛选器 + 卡片列表，但内容标题质量差 |
| science.html | ✅ 完整 | 5 章结构，文案质量高 |
| skills.html | ⚠️ 问题较大 | 展示的是实际安装的技能（15个），而非品牌化技能包 |

### 2.2 index.html 分析

**已有：**
- Hero 区（H1 + 副标题 + 双按钮）✅
- 日记轮播（carousel，3条/slide，自动轮播）✅
- 6 个统计数字卡片 + CountUp 动画 ✅
- 4 大能力 Tab 切换展示 ✅
- 技能商店 3×2 预览网格 ✅
- 底部 CTA 区 ✅

**缺失：**
- ❌ 无插画/图片（Hero 区、轮播区全是纯文字）
- ❌ 轮播日记无图（sanwan.ai 每条有大图封面）
- ❌ 无"领养/下载产品"CTA（Larry 暂无对外产品，但缺乏商业转化路径）
- ❌ 统计数字卡片是纯数字，没有媒体效应/故事梗概分栏
- ❌ 无社媒/外链入口（GitHub, Twitter/X, 公众号）

### 2.3 diary.html 分析

**已有：**
- 标签筛选器（全部/产品/技术/内容/自动化/多Agent/RAG/部署）✅
- 日记卡片列表（按日期倒序）✅
- 最新一条有插画（2026-03-08 带插画 ✅，其余无图）
- 自动同步脚本（cron）✅

**缺失：**
- ❌ 大部分日记是"Conversation Summary"标题——无叙事感，质量差
- ❌ 大部分日记无插画
- ❌ 无搜索功能（只有标签筛选）
- ❌ 无日记详情页（点击日记没有跳转，只是卡片展示）
- ❌ 无锚点链接（diary.html#day7 这种深度链接无法实现）

### 2.4 science.html 分析

**已有：**
- 5 章完整结构 ✅
- 文案质量高（Larry 金句、场景故事） ✅
- 章节结构清晰 ✅

**缺失：**
- ❌ 无动态数字动画（sanwan.ai 科普页第三章有 CountUp）
- ❌ 无图表/视觉对比（纯文字，无 ChatGPT vs Agent 对比表格）
- ❌ 无留言板

### 2.5 skills.html 分析

**严重问题：**
- ⚠️ 展示内容错位：当前展示的是 OpenClaw 安装的实际技能（anti-crawl-fetch、baoyu-cover-image 等），而不是 COPY.md 里品牌化的"技能包"（内容全链路、竞品情报雷达等）
- ❌ 每个技能描述是英文/技术描述，不是面向用户的中文卖点文案
- ❌ 无技能分类标签
- ❌ 无技能详情页（点击无反应）
- ❌ 技能数量展示是"clawhub install xxx"命令，不是用户友好的按钮

### 2.6 整体技术审计

| 维度 | 评分 | 说明 |
|------|------|------|
| 响应式设计 | 7/10 | 有 media query，移动端基本可用 |
| JS 交互 | 7/10 | 有轮播、Tab 切换、筛选、CountUp |
| 动效水平 | 5/10 | hover 效果、轮播动画，但缺乏进入动画/滚动触发 |
| SEO | 6/10 | 有 meta title/description，但缺 og:image、结构化数据 |
| 视觉冲击力 | 5/10 | 纯文字为主，无图片内容 |
| 内容质量 | 6/10 | 主页文案好，但日记内容粗糙 |
| 社区/互动 | 0/10 | 完全没有社区互动模块 |
| 商业转化 | 3/10 | CTA 存在，但无实际转化路径 |
| 多语言 | 0/10 | 无 |
| 技能商店 | 3/10 | 内容错位，无用户体验 |

## 三、差距分析

### 3.1 功能模块差距

| 功能 | sanwan.ai | agent-site | 差距描述 |
|------|-----------|------------|---------|
| 日记大图轮播 | ✅ 每条带插画 | ⚠️ 只有1条有图 | 视觉冲击力差距巨大 |
| 日记详情页 | ✅ 每条可点击 | ❌ 无 | 无法深阅读 |
| 技能详情页 | ✅ 每个技能独立页 | ❌ 无 | 技能无法深展示 |
| 文章页 | ✅ 有 articles.html | ❌ 无 | 缺少深度内容载体 |
| 产品领养页 | ✅ easyclaw.html | ❌ 无 | 缺商业转化入口 |
| 留言板 | ✅ giscus | ❌ 无 | 无社区互动 |
| 多语言 | ✅ EN/JA/DE | ❌ 无 | 无法触达国际用户 |
| 技能筛选标签 | ✅ 官方/必装/热门/社区 | ❌ 无 | 技能无法分类浏览 |
| 深度链接锚点 | ✅ #day6 等 | ❌ 无 | 无法精确引用 |
| 社媒外链 | ✅ GitHub/X/EasyClaw | ⚠️ 只有页面链接 | 缺乏社交证明 |

### 3.2 内容质量差距

| 维度 | sanwan.ai | agent-site | 差距 |
|------|-----------|------------|------|
| 日记标题 | 故事性强（"老板过生日，AI独自值守"） | 大量"Conversation Summary" | 内容运营缺位 |
| 技能文案 | 中文用户友好，带卖点 | 英文技术描述 | 定位错位 |
| 插画覆盖率 | 100%（每条日记有图） | ~7%（只有1条有图） | 视觉内容严重不足 |

### 3.3 交互体验差距

- sanwan.ai 轮播有**左右翻页按钮**，agent-site 只有圆点导航（移动端体验差）
- sanwan.ai 日记可点击进详情，agent-site 日记卡片点击无反应
- sanwan.ai 技能有独立页（40+），agent-site 技能无详情

### 3.4 视觉效果差距

- sanwan.ai：大图轮播 + 插画 + 媒体效应数据卡片，视觉层次丰富
- agent-site：几乎纯文字 + 数字，缺乏视觉锚点和情感连接

### 3.5 技术架构差距

- sanwan.ai：每个日记/技能有独立 HTML 页（利于 SEO，有锚点）
- agent-site：单页展示，无独立子页，无锚点

### 3.6 SEO/性能差距

- sanwan.ai：独立页面多 = 多个 SEO 落地页
- agent-site：缺 `og:image`、缺结构化数据（JSON-LD）、缺 sitemap
