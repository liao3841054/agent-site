### 4大核心能力展示

#### 能力一：Context Engineering

**标题：** 你写的 Prompt 是废料，他的才叫燃料。
**副标题：** 一样的模型，结果差十倍。差的不是 AI，是上下文工程。

**场景故事：**

🔬 **场景 1 — 从废话到精准**
一个客服系统，原来的 Prompt 写了 500 字，AI 还是答非所问。Larry 重写了上下文结构——角色定义、约束边界、输出格式、示例锚点——四层下去，客诉率下降了 60%。

📊 **场景 2 — 让 AI 知道它不知道什么**
RAG 系统上线初期，AI 经常"编"答案。加了一层"知识边界声明"的上下文，AI 开始主动说"这个问题我的知识库里没有"。幻觉问题，是上下文没给够，不是模型太蠢。

⚡ **场景 3 — 上下文即系统**
产品参谋长每次启动，都自动加载：竞品库、用户画像、历史决策记录。不用每次重新解释背景。Larry 的原话：上下文不是 Prompt 的附件，上下文就是系统本身。

**引用语：**
> "大多数人在调教 AI，我在设计系统。区别是：调教是一次性的，系统是可以跑的。" — Larry

**副标题：** 没有 RAG 的 AI 是在背课文，有 RAG 的 AI 才是在查档案。

**场景故事：**

📁 **场景 1 — 公司知识库上线**
一家 50 人公司，制度文档散落在飞书、钉钉、本地硬盘。Larry 用 RAG 把它们统一索引，员工问任何内规，AI 直接引用原文作答，附带页码。HR 主管说：这比我还懂公司制度。

🔍 **场景 2 — 竞品情报实时更新**
产品参谋长每天爬取竞品更新，存入向量库。Larry 问"竞品昨天有什么动作"，AI 给的是当天数据，不是三个月前的训练集。情报不过夜，决策不滞后。

📰 **场景 3 — 行业报告秒变可用洞察**
一份 80 页的行业报告，人读要 3 小时，还不一定抓到重点。RAG 索引后，直接问"对我们的影响是什么"，30 秒出 5 条定制化结论。报告的价值，在于被用上，不在于被收藏。

**引用语：**
> "AI 的智商不是问题，记忆力才是。RAG 就是给 AI 装上了真正的大脑。" — Larry

### 第一章：什么是 AI Agent？

**章节标题：** Agent 不是聊天机器人。

人们以为 AI 就是能聊天的程序。错了。

聊天机器人是应答式的：你问，它答，结束。
Agent 是行动式的：你说目标，它拆解任务，调用工具，自主执行，反馈结果。

两者的区别，就是接线员和员工的区别。

**Larry 金句：**
> "ChatGPT 是个聪明的外包，Agent 是个能独立完成工作的同事。你想要的是哪个？"

**Agent 的核心能力：**
- **感知**：读文件、搜网络、看数据
- **思考**：拆解任务、制定计划、判断优先级
- **行动**：调 API、写代码、发消息、存数据
- **反馈**：报告结果、处理异常、等待下一个指令

Larry 的技术专家⚙️ 就是这样工作的。你说"把这份报告部署到服务器"，它自己找 GitHub、下载、配置、测试、上线。你不需要教每一步。

---

### 第二章：什么是 Skill（技能包）？

**章节标题：** 技能是 Agent 的超能力模块。

一个空白的 Agent 什么都做不了。
就像一个聪明的新员工，没有培训，也交不出活。

Skill 就是给 Agent 的技能培训包。

每个 Skill 包含：
- **这个技能用来干什么**（定义）
- **遇到什么情况触发**（触发条件）
- **用哪些工具来执行**（工具调用）
- **怎么算做完了**（完成标准）

Larry 的笔杆子✍️有个叫 `baoyu-post-to-wechat` 的 Skill，触发条件是"文章写完且审核通过"，执行步骤是"格式化→上传封面图→发布→回传链接"，做完自动通知 Larry。

**Larry 金句：**
> "Skill 是知识的固化。你教了一次，它记住了，以后永远这样干。人类员工会忘，Skill 不会。"

你不需要每次都重新教 Agent 怎么做同一件事。封装成 Skill，一劳永逸。

---

### 第三章：什么是 RAG？

**章节标题：** AI 说假话，不是它坏，是它不知道真相。

语言模型有个根本缺陷：训练截止日期之后的事它不知道，你自己的私有数据它更不知道。

你问它你公司的报销流程，它只能编一个听起来合理的答案。

RAG（检索增强生成）就是解决这个问题的。

工作原理很简单：
1. 把你的文档全部向量化，建立索引
2. 用户提问时，先检索相关文档片段
3. 把检索到的真实内容+用户问题一起送给 AI
4. AI 基于真实数据作答，不再靠猜

**效果是什么：** AI 说的每句话都有原文依据。它会说"根据你的第 37 号内部文件……"，而不是"我认为……"

Larry 为产品参谋长🔍搭建的竞品情报 RAG，每天自动更新，实时检索，再也不用担心 AI 在用过时的信息。

**Larry 金句：**
> "不上 RAG 的 AI 系统，就是把一个失忆症患者放在客服岗位上。他很聪明，但他不知道你公司发生了什么。"

---

### 第四章：什么是多 Agent 团队？

**章节标题：** 一个 AI 是助手，多个 AI 是组织。

单个 Agent 有局限：它的注意力有限，它的专业深度有限，它同时处理的任务数量有限。

就像一个全能员工，再厉害也做不到同时写代码、做分析、发内容。

多 Agent 团队的逻辑：分工。

Larry 的团队架构：
- **总指挥（Larry）**：定目标，做最终决策
- **产品参谋长🔍**：负责分析、洞察、竞品、用户
- **技术专家⚙️**：负责代码、部署、系统、自动化
- **笔杆子✍️**：负责内容、文案、发布、传播

三个 Agent 并行工作，各有专属 Skill，通过任务编排系统交接工作，Larry 只需要在关键节点做决策。

**这不是科幻，这是工程问题。**

每个 Agent 的边界要清晰，接口要定义好，结果要可验证。管 AI 团队，和管人类团队，本质没什么不同。就是更快，更便宜，不请假。

**Larry 金句：**
> "AI 团队不是一个工具，是一套组织架构。架构设计得好，每多加一个 Agent，能力就指数级上升。"

---

### 第五章：什么是 Context Engineering？

**章节标题：** 为什么同一个 AI，别人用得一塌糊涂，你用得行云流水？

不是你更聪明。是你的上下文设计得更好。

Context Engineering（上下文工程）是 Larry 认为 2024 年最被低估的 AI 技能。

它包含：
- **系统角色定义**：告诉 AI 它是谁、它的边界在哪
- **任务框架设计**：把复杂任务拆成 AI 能执行的步骤
- **知识注入策略**：什么时候注入什么背景信息
- **输出格式约束**：让 AI 输出你能直接用的结构
- **记忆管理机制**：长对话中如何保持上下文不"忘事"

大多数人写 Prompt，只写了第一层：告诉 AI 要做什么。

Larry 写的是系统：五层上下文，每一层都有设计，每一层都有意图。结果是，他的 AI 团队第一次收到任务就能干对，而不是来回磨十几遍。

**Larry 金句：**
> "Prompt 是一次性的对话，Context 是持续运行的系统。大多数人还在写对话，我在设计系统。"

### 技能包卡片（8个）

---

#### 技能包 1：内容全链路发布

**所属成员：** 笔杆子✍️
**标签：** 内容创作 · 自动化

**描述：**
从选题到发布，一条龙搞定。公众号文章写完自动生成封面图，自动格式化排版，一键发布到微信。支持同步到 X/Twitter Thread，适配小红书图文格式。笔杆子一个人干完原来内容团队三个人的活。

**包含技能：**
- `baoyu-url-to-markdown` — 网页抓取转写作素材
- `baoyu-markdown-to-html` — Markdown 转公众号 HTML
- `baoyu-cover-image` — 自动生成文章封面图
- `baoyu-post-to-wechat` — 一键发布到微信公众号
- `baoyu-post-to-x` — 发布到 X/Twitter Thread
- `baoyu-slide-deck` — 内容转演示幻灯片

**技能数：** 6个技能

---

#### 技能包 2：竞品情报雷达

**所属成员：** 产品参谋长🔍
**标签：** 竞品分析 · 实时监控

**描述：**
7×24 自动爬取竞品官网、产品更新日志、App 应用商店评论、社交媒体动态。每天自动生成情报摘要，推送到飞书或你指定的频道。参谋长睡觉时也在盯着你的竞争对手。

**包含技能：**
- `competitor-monitor` — 竞品动态实时监控
- `review-analyzer` — App 评论情感分析
- `market-brief` — 每日市场情报摘要生成
- `trend-radar` — 行业趋势关键词追踪

**技能数：** 4个技能

---

#### 技能包 3：RAG 知识引擎

**所属成员：** 技术专家⚙️
**标签：** RAG · 知识管理

**描述：**
把你的内部文档、制度手册、会议记录、历史项目变成可检索的私有知识库。支持飞书文档、Notion、本地 PDF、Word 多源接入。员工问任何内规，AI 直接引用原文，附带页码来源。

**包含技能：**
- `doc-ingestion` — 多格式文档向量化入库
- `feishu-sync` — 飞书文档自动同步索引
- `rag-query` — 带来源引用的精准问答
- `knowledge-update` — 知识库自动增量更新
- `search-audit` — 检索命中率监控与优化

**技能数：** 5个技能

---

#### 技能包 4：Code Agent 交付流水线

**所属成员：** 技术专家⚙️
**标签：** Code Agent · 自动化开发

**描述：**
从需求描述到代码提交，技术专家全程接管。自动写代码、自动运行测试、自动修 bug、自动提 PR，遇到无法解决的问题才会叫醒你。Larry 睡着的时候，技术专家已经提了三个 PR。

**包含技能：**
- `code-generate` — 基于需求自动生成代码
- `test-runner` — 自动运行测试套件
- `bug-fixer` — 测试失败自动分析修复
- `pr-submitter` — 自动提交 Pull Request
- `code-review` — 代码质量自动审查
- `dep-updater` — 依赖库安全更新
- `deploy-checker` — 部署后健康检查

**技能数：** 7个技能

---

#### 技能包 5：用户洞察提炼

**所属成员：** 产品参谋长🔍
**标签：** 用户研究 · 产品决策

**描述：**
200 条用户反馈，人工整理要一天，参谋长 30 分钟出结论。支持问卷数据、用户访谈录音转写、App 评论批量分析。输出格式直接可用：问题分类 + 优先级排序 + 行动建议。

**包含技能：**
- `feedback-cluster` — 用户反馈聚类分析
- `interview-digest` — 访谈录音转写+提炼
- `persona-builder` — 用户画像自动生成
- `priority-matrix` — 需求优先级矩阵输出

**技能数：** 4个技能

---

#### 技能包 6：本地部署全家桶

**所属成员：** 技术专家⚙️
**标签：** 本地部署 · 安全 · 系统工程

**描述：**
敏感数据不出网的完整解决方案。Ollama 本地模型 + systemd 守护进程 + Cloudflare 边缘路由，一套跑通。服务器重启自动拉起，崩溃自动重启，你出差旅行 AI 团队照常运转。

**包含技能：**
- `ollama-setup` — 本地 LLM 环境配置
- `systemd-deploy` — Agent 服务守护进程配置
- `cloudflare-routing` — Cloudflare Workers 边缘路由
- `ssl-auto-renew` — SSL 证书自动续签
- `health-monitor` — 服务健康状态监控
- `backup-restore` — 自动备份与恢复机制
- `log-aggregation` — 多 Agent 日志聚合分析
- `resource-guard` — 服务器资源占用保护

**技能数：** 8个技能

---

#### 技能包 7：多 Agent 任务编排

**所属成员：** 总指挥（协调层）
**标签：** 任务编排 · 多Agent · 工作流

**描述：**
一个指令，多个 Agent 并行工作，结果自动汇总。支持顺序执行、并行执行、条件分支，任何环节失败自动重试或降级。Larry 管三个 Agent 的方式，不是一个个发命令，是设计工作流。

**包含技能：**
- `workflow-designer` — 可视化任务编排配置
- `parallel-dispatch` — 并行任务分发与汇总
- `retry-handler` — 失败自动重试与降级
- `progress-tracker` — 任务进度实时追踪
- `handoff-protocol` — Agent 间工作交接协议

**技能数：** 5个技能

---

#### 技能包 8：Context Engineering 工具箱

**所属成员：** 总指挥（系统层）
**标签：** Prompt 工程 · 系统设计

**描述：**
不是 Prompt 模板，是上下文系统设计方法论。包含 Larry 3 年实战提炼的：角色定义框架、知识注入策略、输出格式约束、记忆管理机制。拿来就能用，不需要再踩坑。

**包含技能：**
- `role-architect` — AI 角色系统设计工具
- `context-injector` — 动态知识注入策略
- `output-schema` — 结构化输出格式设计
- `memory-manager` — 长对话记忆管理机制
- `hallucination-guard` — 幻觉检测与防护机制
- `eval-suite` — 上下文效果自动评估

**技能数：** 6个技能
