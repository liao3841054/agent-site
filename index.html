4 大能力，构成完整的 AI 团队

每一项能力都有真实场景，每一个场景都发生过。

你写的 Prompt 是废料，他的才叫燃料。
一样的模型，结果差十倍。差的不是 AI，是上下文工程。
🔬 场景 1 — 从废话到精准

一个客服系统，原来的 Prompt 写了 500 字，AI 还是答非所问。Larry 重写了上下文结构——角色定义、约束边界、输出格式、示例锚点——四层下去，客诉率下降了 60%。

📊 场景 2 — 让 AI 知道它不知道什么

RAG 系统上线初期，AI 经常"编"答案。加了一层"知识边界声明"的上下文，AI 开始主动说"这个问题我的知识库里没有"。幻觉问题，是上下文没给够，不是模型太蠢。

⚡ 场景 3 — 上下文即系统

产品参谋长每次启动，都自动加载：竞品库、用户画像、历史决策记录。不用每次重新解释背景。Larry 的原话：上下文不是 Prompt 的附件，上下文就是系统本身。

"大多数人在调教 AI，我在设计系统。区别是：调教是一次性的，系统是可以跑的。"

— Larry
让 AI 说真话的方法只有一个：给它看真实的资料。
没有 RAG 的 AI 是在背课文，有 RAG 的 AI 才是在查档案。
📁 场景 1 — 公司知识库上线

一家 50 人公司，制度文档散落在飞书、钉钉、本地硬盘。Larry 用 RAG 把它们统一索引，员工问任何内规，AI 直接引用原文作答，附带页码。HR 主管说：这比我还懂公司制度。

🔍 场景 2 — 竞品情报实时更新

产品参谋长每天爬取竞品更新，存入向量库。Larry 问"竞品昨天有什么动作"，AI 给的是当天数据，不是三个月前的训练集。情报不过夜，决策不滞后。

📰 场景 3 — 行业报告秒变可用洞察

一份 80 页的行业报告，人读要 3 小时，还不一定抓到重点。RAG 索引后，直接问"对我们的影响是什么"，30 秒出 5 条定制化结论。报告的价值，在于被用上，不在于被收藏。

"AI 的智商不是问题，记忆力才是。RAG 就是给 AI 装上了真正的大脑。"

— Larry
云端 AI 很好，但有些数据，不能出门。
能跑在你自己服务器上的 AI，才是真正属于你的 AI。
🔒 场景 1 — 敏感数据不出网

客户合同、用户隐私数据、内部财务报告——这些东西送给 OpenAI 的 API 是违规的。Larry 用 Ollama 本地跑 Qwen，法务数据分析全程在内网，数据主权在自己手里。

⚙️ 场景 2 — systemd 让 Agent 永不下线

技术专家、笔杆子、产品参谋长，三个 Agent 全部用 systemd 部署成系统服务。服务器重启自动拉起，崩溃自动重启，Larry 旅行时不用担心 Agent 死掉。7×24，不是口号，是配置。

🚀 场景 3 — Cloudflare 做流量入口

agent.nanopromptlab.com 背后是一套完整的部署架构：Cloudflare Workers 做边缘路由，VPS 跑主服务，域名解析 + SSL 自动续签。用户访问感觉丝滑，Larry 自己管得住，不依赖任何第三方平台。

"SaaS 是租房，本地部署是买房。租的永远是别人的，部署到自己服务器，才叫落地。"

— Larry
一个人指挥，三个 Agent 干活，效率是团队的十倍。
不是让 AI 更聪明，是让多个 AI 分工协作。
🎯 场景 1 — 一个指令，三个并行任务

Larry 说："出一份新产品的完整上线方案。"产品参谋长做竞品分析，技术专家写技术规格，笔杆子起草推广文案——三个任务并行，45 分钟后汇总。一个人说话，三个方向同时动。

🔄 场景 2 — Agent 互相传球

笔杆子写完文章，自动传给产品参谋长做事实核查，确认数据准确后再传给技术专家发布上线。每个 Agent 只做自己擅长的，交接点由编排系统控制，Larry 不用手动转发任何文件。

📈 场景 3 — 错误触发重试机制

一次任务编排中，技术专家遇到 API 超时。编排系统自动触发重试，同时通知 Larry。Larry 睡着了，凌晨 4 点任务自动完成，早上醒来收到完成报告。失败不是终点，是触发器。

"管理 AI 团队和管理人类团队没什么不同：分工要清，接口要明，结果要可验证。"

— Larry
